Overview

What is Snapblox?

A fully autonomous multi-agent system that turns game specs into playable Roblox games, hands-free.

⚙

Spec to Playable Game

Write a plain-text spec describing your Roblox game idea. Snapblox handles everything else — architecture, code, assets, and testing — without you writing a single line of Luau.

●

Nightly Build Cycle

Agents fire every night on a schedule. By morning, new features are implemented, PRs are open, and a digest report summarizes exactly what was built and what comes next.

◆

Multi-Agent Pipeline

Specialized agents handle each phase: Architect plans, Planner sprints, Builder codes, QA validates, Reporter digests. Each agent has strict scope — no stepping on toes.

Process

How It Works

Six steps from idea to working Roblox game — fully automated every night.

01
✎

Write a Game Spec

Describe your Roblox game idea in a structured Markdown spec file. Include mechanics, goals, and constraints.

02
🛠

Architect Plans

The Architect agent reads your spec and decomposes it into milestones, tasks, and a dependency graph stored in plan.md.

03
📅

Planner Sprints

Each night, the Planner selects the highest-priority tasks from the plan and generates a structured sprint with assignments.

04
💻

Builder Codes

Builder implements Luau scripts directly inside Roblox Studio via the MCP server — live, in the actual editor.

05
✓

QA Validates

The QA agent reviews each PR against a 26-rule Luau lint checklist and playtest criteria before approving.

06
☼

Reporter Digests

A morning digest lands at 5 am summarizing overnight progress, open PRs, blockers, and tonight's plan.

Agents

Agent Pipeline

Each agent owns a specific role. No agent can override another's domain without a human override.

Agent 1

Architect

  • Reads game spec
  • Produces milestone plan
  • Maps task dependencies
  • Writes plan.md
Agent 2

Planner

  • Generates nightly sprints
  • Assigns tasks to workers
  • Replans on failures
  • Monitors human overrides
Agent 3

Builder

  • Implements Luau features
  • Writes via Roblox Studio MCP
  • Opens GitHub PRs
  • Writes progress log
Agent 4

QA

  • Validates every PR
  • Runs 26-rule lint check
  • Executes playtest eval
  • Labels qa-approved / failed
Agent 5

Reporter

  • Generates morning digest
  • Summarizes overnight work
  • Flags blockers & cost usage
  • Previews tonight's plan
Agent 6

Market Researcher

  • Weekly Roblox market scans
  • Competitor mechanic analysis
  • Trending genre reports
  • New game idea proposals

Getting Started

Up and running in minutes

Clone the repo, create a spec, and launch your first nightly build cycle.

bash 
git clone https://github.com/DevMello/roblox-studio-agency
cd roblox-studio-agency

# 1. Create your first game spec
./scripts/new-game.sh my-game

# 2. Edit games/my-game/spec.md
#    Describe your game: mechanics, goals, constraints

# 3. Launch the night cycle
./scripts/launch-night-cycle.sh

# Check your morning digest tomorrow at 5 am
# reports/morning/$(date +%Y-%m-%d).md

See the full setup guide and configuration reference in the GitHub repository.
